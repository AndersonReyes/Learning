Text and Semantic Elements

Headings (h1-h6)

Headings describe the document's outline, not font size. Use them in order (don't skip from <h1> to <h3>) and have only one <h1> per page.

This is an h3

This is an h4

This is an h5
This is an h6

Paragraphs and inline emphasis

This is a normal paragraph. It contains strong text (bold + semantically important) and emphasized text (italic + stressed meaning).

Compare with bold (b) and italic (i) — these apply the same visual style but carry no semantic meaning. Prefer <strong>/<em> when the emphasis matters to the meaning of the sentence.

Line breaks and thematic breaks

First line of a haiku
Second line continues here
Third line ends it now

The <hr> above marks a thematic break between unrelated sections of content.

Blockquote

The HTML <blockquote> Element indicates that the enclosed text is an extended quotation.

Code and preformatted text

Use <code> for short inline code snippets like const x = 1;. 

function greet(name) {
  return `Hello, ${name}!`;
}

<pre> preserves whitespace and line breaks exactly as written — useful for code blocks or ASCII art.
