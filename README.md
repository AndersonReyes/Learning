# Learning

Personal learning notes, examples, and exercises.

## A note on how this works

This repo is built around how I personally like to learn — no hand-holding.
Each topic gives you notes + runnable examples for context, then a set of
hard exercises with a test suite as the spec (no solution files). You're
expected to research, explore, and arrive at your own implementation; the
tests just define "done," not "how."

Exercises are hard **by default** — I'm a senior engineer, so there's no
ramp-up through basic syntax drills, and that's intentional, not an
oversight.

Clone it, fork it, rip out what doesn't fit. If this approach works for you,
great — if not, it's a free starting point to build something that does.

- [`javascript/`](./javascript) — JavaScript track: notes, runnable examples,
  and test-driven exercises. See [`javascript/README.md`](./javascript/README.md)
  to get started and [`javascript/ROADMAP.md`](./javascript/ROADMAP.md) for the
  full curriculum.
- [`html/`](./html) — HTML track: notes and viewable examples (open
  `.html` files directly in a browser). See [`html/README.md`](./html/README.md).
- [`css/`](./css) — CSS track: notes and viewable examples (open `.html`
  files directly in a browser). See [`css/README.md`](./css/README.md) and
  [`css/ROADMAP.md`](./css/ROADMAP.md).
